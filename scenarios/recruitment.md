# 企业客户成功经理候选人初筛

> 完整的 Amazon Quick 操作步骤参见 [Training Hands-on User Guide](../user-guide.md)。本页仅说明业务场景，不重复 Step-by-step 流程。

## 场景背景

公司计划在上海招聘企业客户成功经理，需要根据岗位要求、候选人资料和初筛政策形成一致、可追溯的初筛建议。

Skill 应区分明确不满足门槛、关键证据缺失和满足门槛的候选人，并避免使用受保护属性或无证据推断。

## Skill 目标

- 检查工作资格、相关经验、语言、出差和入职日期等岗位门槛。
- 基于原始字段评估岗位匹配度，并标记证据来源。
- 区分明确不符合与信息待澄清，不用评分覆盖门槛结果。
- 输出供 HR、面试官和用人经理复核的初筛建议。

## 预期结果

| 候选人 | 预期处理 |
|---|---|
| `C001` | 进一步人工比较 |
| `C002` | 暂不进入下一轮 |
| `C003` | 待澄清后决定 |
| `C004` | 暂不进入下一轮 |

## 人工决策边界

- Skill 不得自动淘汰候选人、发送面试邀请或作出录用决定。
- 年龄、性别、民族、婚育状况和照片等受保护属性不得参与判断。
- 缺失信息必须标记为“未知”或“待澄清”，不得补造候选人经历。
- 最终面试和录用决定由 HR、面试官和用人经理共同完成。
