# 渠道合作伙伴初筛

> 完整的 Amazon Quick 操作步骤参见 [Training Hands-on User Guide](../user-guide.md)。本页仅说明业务场景，不重复 Step-by-step 流程。

## 场景背景

公司计划在华东和华南招募企业软件渠道合作伙伴，需要核查企业资质、客户覆盖、售前交付能力、合规情况和利益冲突。

Skill 应将准入门槛与能力评分分开，并明确标记未经验证的合作伙伴自述和材料缺口。

## Skill 目标

- 检查资质、诚信记录、客户案例、人员配置、政策接受情况和竞争冲突。
- 评估市场覆盖、售前、交付、经营稳定性和合作投入。
- 区分明确违反准入条件与关键材料待澄清。
- 输出供渠道、销售、交付、财务和法务复核的准入建议。

## 预期结果

| 合作伙伴 | 预期处理 |
|---|---|
| `P001` | 优先进入商务洽谈 |
| `P002` | 暂不进入商务洽谈 |
| `P003` | 待澄清后决定 |
| `P004` | 暂不进入商务洽谈 |

## 人工决策边界

- Skill 不得批准签约、授予渠道资格或接受合规风险。
- 承诺商机金额不能替代准入条件和能力证据。
- 合作伙伴自述不得视为已验证事实，缺失信息必须标记为“未知”或“待澄清”。
- 最终签约和渠道授权由销售、渠道、交付、财务和法务共同批准。
